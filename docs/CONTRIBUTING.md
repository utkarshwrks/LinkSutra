We follow the [Contributor Covenant](https://www.contributor-covenant.org/).

---

## 🛠️ How to Contribute

### Reporting Bugs
- Check if the issue already exists in [GitHub Issues](https://github.com/arpitsharma7777/linksutra/issues)
- If not, open a new issue with a clear title and description
- Include steps to reproduce the bug

### Suggesting Features
- Open a GitHub Issue with the `enhancement` label
- Describe the feature and why it would be useful

### Contributing Code
1. Fork the repository
2. Create a new branch from `dev`
3. Make your changes
4. Open a Pull Request to `dev` branch

---

## 🌿 Branch Strategy
```
main     → stable production code only
dev      → all features merge here first
feature/ → your individual work
```

**Always branch from `dev`, never from `main`:**
```bash
git checkout dev
git pull origin dev
git checkout -b feature/your-feature-name
```

---

## ✍️ Commit Message Format

Follow **Conventional Commits**:
```
feat:     new feature
fix:      bug fix
docs:     documentation changes
style:    CSS / UI changes
refactor: code cleanup, no logic change
test:     adding tests
chore:    config, dependencies
```

**Examples:**
```bash
git commit -m "feat: add link click analytics endpoint"
git commit -m "fix: correct login token expiry bug"
git commit -m "docs: update installation steps in README"
```

---

## 🔁 Pull Request Process

1. Make sure your code runs without errors
2. Update README or docs if needed
3. Open PR to `dev` branch — not `main`
4. Write a clear PR description — what changed and why
5. Wait for review — we aim to review within 24 hours

## 💻 Development Setup

**Backend:**
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```

**Frontend:**
```bash
cd frontend
npm install
npm run dev
```

**API Docs** (auto-generated by FastAPI):
```
http://localhost:8000/docs
```
